## コードを書く前に設計を考える

プログラムを作るとき、いきなりコードを書き始めるより**「何をどのように作るか」を先に考える**ことが大切です。この章では、2048を作るための設計を考えます。

---

## ボードのデータ構造

2048のボードは4×4のマス目です。これをプログラムで表現するには、**2次元配列（配列の中に配列が入っている形）** を使います。

```js
[
  [0, 0, 0, 0],
  [0, 0, 2, 0],
  [0, 4, 0, 0],
  [0, 0, 0, 2],
]
```

- 各要素はそのマスのタイルの数字を表します
- `0` は空きマスを表します
- 行（横）を外側の配列、列（縦）を内側の配列で表します

この配列を `board` という名前の状態として管理します。

---

## ゲームの状態一覧

このゲームで管理する状態は次の3つです。

| 状態名 | 型 | 説明 |
|---|---|---|
| `board` | 二次元配列（4×4） | ボードの現在の数字の並び |
| `score` | 数値 | 現在のスコア |
| `gameOver` | 真偽値 | ゲームオーバーかどうか |

---

## 実装する機能の一覧

2048に必要な機能を整理します。

### UIコンポーネント

| コンポーネント | 役割 |
|---|---|
| `App` | ゲーム全体の管理（状態・キー入力） |
| `Board` | 4×4のグリッドを表示 |
| `Tile` | 各マスの数字を表示 |

### ゲームロジック（関数）

| 関数 | 役割 |
|---|---|
| `createEmptyBoard()` | すべて0の空ボードを作る |
| `addRandomTile(board)` | 空きマスにランダムで2か4を追加する |
| `moveLeft(board)` | タイルを左に移動・合体させる |
| `moveRight(board)` | タイルを右に移動・合体させる |
| `moveUp(board)` | タイルを上に移動・合体させる |
| `moveDown(board)` | タイルを下に移動・合体させる |
| `isGameOver(board)` | ゲームオーバーかどうか判定する |

## ファイル構成

ゲームロジックとUIを分けて管理します。

```
src/
├── App.jsx          # メインコンポーネント（状態管理・キー入力）
├── App.css          # スタイル
├── components/
│   ├── Board.jsx    # ボードコンポーネント
│   └── Tile.jsx     # タイルコンポーネント
└── utils/
    └── gameLogic.js # ゲームロジックの関数
```

`components/` フォルダにUIコンポーネント、`utils/` フォルダにゲームロジックをまとめることで、コードが整理されます。

フォルダを作成しておきましょう。

```bash
mkdir -p src/components src/utils
```

`-p` オプションは「途中のフォルダが存在しない場合も含めてまとめて作る」という意味です。たとえば `src` フォルダがまだない状態でも、`src/components` と `src/utils` を一度に作れます。また、すでにフォルダが存在してもエラーにならないため、何度実行しても安全です。
