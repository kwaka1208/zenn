本書では、プログラミングを始めるために不可欠な3つのツール、**Git**、**Node.js**、**Python** のインストール方法を解説します。あわせて、開発環境を快適に整えるための **EditorConfig**、**PATH の設定**、**make コマンド** についても取り上げます。

Windows、macOS、Linux (Ubuntu想定) の3つの OS に対応しており、それぞれの環境に最適な方法を紹介していますので、お使いのプラットフォーム、利用するツールのページを適宜参照してください。

## 対象とする OS
本書では、以下の OS 環境を対象としています。

- **Windows**:
  Windows の手順については、以下の2つの環境を呼び分けて解説します。
  - **WSL (Windows Subsystem for Linux)**: Windows 上で Linux (Ubuntu) を動作させる環境です。
  - **Windows ネイティブ**: Windows 上で直接 PowerShell を使用してツールを動作させる環境です。
- **macOS**: Apple Silicon Mac
- **Linux**: Ubuntu などの Debian 系ディストリビューション

Windows ユーザーの方は、ご自身が選択した環境に合わせて各章の該当箇所を参照してください。

## WSL かネイティブ環境か、どちらを選ぶべきか？

Windows ユーザーにとって、WSL を使うか、Windows ネイティブ環境を使うかは大きな選択です。以下の基準を参考にしてください。

### 1. WSL を選ぶべきケース（推奨）
**Web開発、AI/データ分析、バックエンド開発を目指す場合**は、WSL が強く推奨されます。

*   **Linux との互換性**: ほとんどの開発用サーバーやクラウド環境は Linux です。WSL を使うことで、本番環境に近い環境で開発でき、「自分の PC では動いたのにサーバーでは動かない」というトラブルを減らせます。
*   **ドキュメントの多さ**: インターネット上の多くの開発チュートリアルは Mac や Linux を前提としています。WSL なら、それらのドキュメントのコマンドをそのままコピー＆ペーストで利用できます。
*   **パッケージ管理の容易さ**: `apt` などの強力なパッケージ管理ツールが使えるため、ツールのインストールや更新が非常にスムーズです。
*   **Docker との親和性**: モダンな開発に必須のツールである Docker は、WSL 2 上で動作させることが標準となっています。

### 2. Windows ネイティブ環境を選ぶべきケース
以下のような特定の目的がある場合は、Windows 上に直接インストールする方がスムーズです。

*   **Windows 専用アプリの開発**: C# や .NET を使った Windows アプリの開発や、Excel 等の Office 連携ツールを作る場合。
*   **学習コストを最小限にしたい**: 「Linux のコマンド（`ls` や `cd` など）を覚えるのが負担」「Windows のファイル管理（Cドライブなど）の感覚のまま作業したい」という初心者の方。
*   **PC のスペックが低い**: WSL は仮想化技術を使うため、メモリ（RAM）をそれなりに消費します。メモリが 8GB 以下の PC では、ネイティブ環境の方が軽快に動作することがあります。

---

**迷った場合は、まずは WSL をインストールしてみることをおすすめします。** 後の章では両方の手順を解説していますので、途中で切り替えることも可能です。

## 構築するツール
1. **PowerShell 7 (Windowsのみ)**: モダンなシェル環境
1. **WSL (Windowsのみ)**: Linux サブシステム（Ubuntu）
1. **Homebrew (macOSのみ)**: パッケージ管理ツール
1. **Git**: バージョン管理システム
1. **Node.js**: JavaScript 実行環境
1. **Python**: プログラミング言語と仮想環境
1. **Visual Studio Code**: エディタ

## 開発環境を整えるための設定・ツール
1. **EditorConfig**: プロジェクト内のコードスタイルをエディタ横断で統一する設定ファイル
1. **PATH の設定**: インストールしたツールをどこからでも呼び出せるようにするための環境変数の設定方法（OS別）
1. **make コマンド**: よく使う作業をコマンド一発で実行できるようにするタスクランナー

Windows ユーザーの方は、まず次の章で WSL をインストールするか、あるいは直接 Windows にツールをインストールするかを選択して進めてください。
