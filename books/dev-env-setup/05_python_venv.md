---
title: "Python 仮想環境の構築"
---

Python で開発を行う際は、プロジェクトごとにライブラリを独立して管理するための「仮想環境」を作成することが推奨されます。これにより、「プロジェクトAではバージョン1.0が必要だが、プロジェクトBではバージョン2.0が必要」といった依存関係の衝突を防ぐことができます。

Python 標準の **`venv`** モジュールを使用する方法を解説します。

## 1. 仮想環境の作成
プロジェクトのディレクトリに移動し、以下のコマンドを実行します。

```bash
# プロジェクトディレクトリを作成して移動
mkdir my-project
cd my-project

# 仮想環境（名前は .venv が一般的）を作成
python3 -m venv .venv
```

## 2. 仮想環境の有効化 (Activate)
作成しただけでは仮想環境は使われません。OS ごとに異なるコマンドで「有効化」する必要があります。

### Windows (PowerShell)
```powershell
.\.venv\Scripts\Activate.ps1
```

### macOS / Linux
```bash
source .venv/bin/activate
```

有効化されると、ターミナルのプロンプトの先頭に `(.venv)` と表示されます。

## 3. ライブラリのインストール
仮想環境が有効な状態で `pip` を使用すると、その環境内だけにライブラリがインストールされます。

```bash
pip install requests
```

## 4. 仮想環境の解除 (Deactivate)
作業を終了する場合は、以下のコマンドで仮想環境を抜けることができます（どの OS でも共通です）。

```bash
deactivate
```

これで、クリーンな開発環境を保つことができます。
