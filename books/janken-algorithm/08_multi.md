## 三人版から多人数版へ

前のチャプターで、`hands_list` と `sum()` を使った三人版のアルゴリズムを作りました。このアルゴリズムは、実はコンピュータが何台に増えても**ほぼそのまま使える**のです。

必要な変更は2点だけです。

1. コンピュータの手を **リスト**でまとめて管理する
2. `hands_list` への登録を **`for` ループ**でまとめる

## コンピュータをリストで管理する

コンピュータが10台になったとき、三人版のやり方だと変数が `cp1` `cp2` `cp3` ... `cp10` と10個必要になります。これを**リスト**（配列）にまとめます。

```python
MAX_COMPUTER = 10  # コンピュータの台数
cp = []            # コンピュータの手を入れるリスト（最初は空）

for i in range(MAX_COMPUTER):
    cp.append(random.randint(0, 2))  # ランダムな手をリストに追加
```

`cp.append(値)` でリストの末尾に値を追加できます。`range(MAX_COMPUTER)` は 0, 1, 2, ..., 9 と繰り返すので、10回ループして10台分の手を決めます。

## hands_list の登録も for ループで

三人版では手動で3行書いていた `hands_list` の登録を、`for` ループでまとめます。

```python
hands_list = [0, 0, 0]
hands_list[player] = 1  # プレイヤーの手

for i in range(MAX_COMPUTER):
    hands_list[cp[i]] = 1  # コンピュータ i 台目の手
```

`cp[i]` でリストの `i` 番目の要素（コンピュータの手）を取得し、`hands_list` に登録します。コンピュータが何台になっても、このループを変える必要はありません。

## 完成したプログラム

```python
import random

hands        = {0: "ぐー", 1: "ちょき", 2: "ぱー"}
MAX_COMPUTER = 10  # コンピュータの台数（この数字を変えれば何台でも対応できる）

player = random.randint(0, 2)
print(f"プレイヤーの手: {hands[player]}")

cp = []
for i in range(MAX_COMPUTER):
    cp.append(random.randint(0, 2))
    print(f"コンピュータ{i + 1}の手: {hands[cp[i]]}")

print("結果: ", end="")

# 出た手の種類をリストで管理する
hands_list = [0, 0, 0]
hands_list[player] = 1

for i in range(MAX_COMPUTER):
    hands_list[cp[i]] = 1

result = sum(hands_list)

if result == 1 or result == 3:
    print("あいこ")
elif player == 0 and hands_list[1] == 1:
    print("勝った")
elif player == 1 and hands_list[2] == 1:
    print("勝った")
elif player == 2 and hands_list[0] == 1:
    print("勝った")
else:
    print("負けた")
```

## プログラムの解説

### コンピュータの台数を変えるには

```python
MAX_COMPUTER = 10
```

この1行の数字を変えるだけで、コンピュータの台数を自由に変えられます。5にすれば5台、100にすれば100台と対戦できます。

### if 文はたった5行

勝ち負けを判定する `if` 文は三人版（改）と全く同じです。コンピュータが増えても `if` 文は増えません。

```python
if result == 1 or result == 3:
    print("あいこ")
elif player == 0 and hands_list[1] == 1:
    print("勝った")
elif player == 1 and hands_list[2] == 1:
    print("勝った")
elif player == 2 and hands_list[0] == 1:
    print("勝った")
else:
    print("負けた")
```

### なぜ if 文が増えないのか

判定に必要な情報は「**どの種類の手が出たか**」だけで、**何人がその手を出したか**は関係ありません。`hands_list` はまさにその「どの種類の手が出たか」を記録しています。

100人で対戦しても、最終的に `hands_list` は `[0か1, 0か1, 0か1]` の3要素のリストであり、判定ロジックは変わりません。

## 実は、人間も同じ順番で判定している

最初のチャプターで「あなたはどうやって勝ち負けを判定している？」という問いを出しました。

改めて振り返ってみると、完成したプログラムの判定の順番はこうです。

1. **あいこかどうか**（全員同じ手 or 三すくみ）を確認する
2. **勝ちかどうか**を確認する
3. それ以外は**負け**

```python
if result == 1 or result == 3:
    print("あいこ")          # ← まずあいこを判定
elif player == 0 and hands_list[1] == 1:
    print("勝った")          # ← 次に勝ちを判定
elif ...
else:
    print("負けた")          # ← それ以外はすべて負け
```

これは、じゃんけんの結果を見た瞬間に人間が無意識でやっていることと同じ順番です。「あいこじゃないな → 自分の手は勝てるかな → 勝てないなら負けだ」という流れで、9通りの組み合わせを全部確認しなくても瞬時に判断できています。

コンピュータが100台になっても、人間がじゃんけんの結果を判断するステップは変わりません。それと同じように、このプログラムの判定ロジックもコンピュータが何台になっても変わりません。

まず全部書いてみることで問題の全体像が見えてきて、そこからパターンに気づいて整理できるようになります。「とにかく全部書く」は、遠回りではなく問題を理解するための大切な一歩です。
