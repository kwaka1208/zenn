---
title: 表で整理するとバグが減る
---

この本では、プログラムを書く前に**じゃんけんの判定表**を作りました。

| プレイヤー | コンピュータ | 結果 |
|:---:|:---:|:---:|
| ぐー（0） | ぐー（0） | あいこ |
| ぐー（0） | ちょき（1） | 勝ち |
| ぐー（0） | ぱー（2） | 負け |
| ちょき（1） | ぐー（0） | 負け |
| ちょき（1） | ちょき（1） | あいこ |
| ちょき（1） | ぱー（2） | 勝ち |
| ぱー（2） | ぐー（0） | 勝ち |
| ぱー（2） | ちょき（1） | 負け |
| ぱー（2） | ぱー（2） | あいこ |

この表があったからこそ、9パターンの `if` 文を**過不足なく**書くことができました。

## 表で整理することの効果

表で整理するメリットは「**抜け漏れを防げる**」ことです。文章で「ぐーはちょきに勝つ」と書くと、書いた本人には分かっても、何パターンあるか・全部書いたかどうかは確認しにくいです。表にすると行と列の組み合わせで**全パターンが一目でわかる**ため、書き忘れに気づきやすくなります。

プログラムを書くときに整理しやすいものの例を挙げます。

| 整理したいもの | 表の行 | 表の列 |
|---|---|---|
| じゃんけんの勝ち負け | プレイヤーの手 | コンピュータの手 |
| 会員種別ごとの料金 | 会員種別 | 通常・割引・無料など |
| エラーコードと対処 | エラーコード | メッセージ・対処方法 |
| 権限と操作の可否 | ユーザー権限 | 操作の種類 |

## ドキュメントも表で書く

仕様書やドキュメントも、文章で書けるものを**あえて表にする**ことをおすすめします。

❌ 文章で書いた例：
> ぐーはちょきに勝ち、ちょきはぱーに勝ち、ぱーはぐーに勝ちます。あいこの場合は同じ手が出たときです。

⭕ 表で書いた例：

| 出した手 | 勝てる相手 |
|:---:|:---:|
| ぐー | ちょき |
| ちょき | ぱー |
| ぱー | ぐー |

文章の場合、読んでいて「全部で何パターンあるのか」が分かりにくく、書いた側も漏れに気づきにくいです。表にすると行数＝パターン数が明確なので、**レビューしやすく、実装との対応も確認しやすい**です。

プログラムを書く前に表を作る習慣をつけると、コードの品質が上がります。「表にできないか？」と考えてみることが、バグの少ないプログラムへの近道です。
