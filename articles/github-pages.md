## はじめに

この記事は、生成AIなどで作成したページやアプリ（HTML単体で動作するもの）を公開したい方向けに、その手段のひとつとして **GitHub Pages** の使い方を紹介するものです。

Gitのようなプログラマーむけのツールをできるだけ使わずに公開できる手順として紹介していますので、プログラミング経験がない方でも安心して読み進めてください。

---

## GitHub Pagesとは？

### GitHubって何？

「GitHub（ギットハブ）」とは、世界中のプログラマーが使う**ファイルの保管・共有サービス**です。

- 作ったファイルをクラウド上に保存できる
- 世界中の人と共有・協力できる
- 無料で使い始められる

### GitHub Pagesとは？

GitHub Pagesは、GitHubに保存したファイルを**そのままWebサイトとして世界中に公開できる無料サービス**です。

- 難しいサーバー設定は一切不要
- ファイルを保存するだけで公開できる
- 自分だけの専用URLが手に入る

### GitHub Pagesの3つのメリット

**1. 完全無料**
サーバー代も公開費用も **0円**です。

**2. ずっと公開され続ける**
自分が削除しない限り、サイトはずっとインターネット上に存在し続けます。

**3. 自分だけのURL**
`https://(ユーザー名).github.io/` というアドレスが手に入ります。

### こんなサイトが作れます

GitHub Pagesはこんな用途によく使われています。

- **自己紹介サイト** — 就職・転職のポートフォリオに
- **学習メモの公開** — 勉強した内容をまとめて発信
- **サークルや団体のサイト** — イベント告知・メンバー紹介
- **プロジェクト集** — 写真やプロジェクトを並べたギャラリー

HTMLとCSSさえ書ければデザインは自由自在ですし、今はAIに任せることもできます。

## 準備するもの

### 必要なのは「GitHubアカウント」だけ！

特別なソフトのインストールは不要です。必要なものはたったひとつ、**GitHubのアカウント（無料）**だけです。

### ユーザー名は慎重に決めよう

GitHub Pagesのサイトアドレスは以下の形式になります。

```
https://(ユーザー名).github.io/...
```

登録時のユーザー名がそのままURLに使われます。あとから変更すると既存のURLが変わってしまうため、**最初に慎重に決めておく**のがおすすめです。

- ✅ 英数字・ハイフン（`-`）が使えます
- ❌ スペース・記号は使えません

### アカウント登録の手順

1. [https://github.com/signup/](https://github.com/signup/) にアクセス
2. Googleアカウント / Appleアカウント / メールアドレスのいずれかで登録
3. 人間であることを確認するためのパズルに挑戦（結構難しいです）
4. メールアドレスで登録した場合は、メールで届いた6桁のコードを入力

![認証メールの例](/images/github-pages/mail.png)

## 3ステップで公開しよう

作業はたった3ステップで完了します。それぞれ5分もあればできます。

- **STEP 1** リポジトリ（フォルダ）を作る
- **STEP 2** `index.html`（ページの中身）を作る
- **STEP 3** 公開設定を確認する

### STEP 1: リポジトリを作る

「リポジトリ」とは、ファイルをまとめる**専用のフォルダ**のことです。まずはこのフォルダを作ります。

1. GitHubにログインし、右上の **[+]** ボタンをクリック
2. **[New repository]** を選択
3. **[Repository name]** に **`(ユーザー名).github.io`** と入力
   - ユーザー名が `kwaka1208` なら `kwaka1208.github.io`
4. **[Public]**（公開）を選ぶ
5. **[Add README]** のスイッチをオンにする
6. **[Create repository]** をクリック

![リポジトリ作成画面](/images/github-pages/repository.png)

#### リポジトリ名のポイント

**`(ユーザー名).github.io`** という名前はGitHubの特別なルールです。この名前のリポジトリが、あなたのサイトの「トップページ」になります。一度作ればOKで、プロジェクトごとに新しいリポジトリを作る必要はありません。

ユーザー名が `kwaka1208` の場合：

| リポジトリ名 | サイトのURL |
|---|---|
| `kwaka1208.github.io` | `https://kwaka1208.github.io/` |

プロジェクトは、このリポジトリの中にフォルダを作って追加していきます。

:::message alert
**リポジトリの公開について**
無料アカウント（GitHub Free）でGitHub Pagesを利用する場合、リポジトリは **Public（公開）** にする必要があります。

Publicリポジトリの内容は、Webサイトとして公開していないファイルも含め、**リポジトリ内のすべてのファイルが誰でも閲覧できる状態**になります。パスワードや個人情報など、他人に見られたくない情報はリポジトリに置かないよう注意してください。
:::

### STEP 2: index.html を作る

リポジトリの中にHTMLファイルを作ります。

1. リポジトリのページで **[Code]** ボタンの左にある **[+]** ボタンをクリック
2. **[Create new file]** を選択
3. ファイル名の欄に `(フォルダ名)/` と入力します。`/` を入力した時点でフォルダ名として認識されます

`sample` まで入力した状態：
![フォルダ名入力途中](/images/github-pages/folder01.png)

`/` を入力した直後：
![フォルダ名確定](/images/github-pages/folder02.png)

4. 続けて `index.html` と入力します

![index.htmlまで入力した状態](/images/github-pages/folder03.png)

#### なぜ「index.html」という名前？

Webサーバーには「フォルダを開いたとき、まず `index.html` を表示する」というルールがあります。

- `https://example.com/` にアクセス → `index.html` が表示される
- `https://example.com/profile/` にアクセス → `profile/index.html` が表示される

このルールを利用することで、URLに `index.html` を付けずにページを作ることができます。

#### index.html の中身を書いてみよう

まずはこれをそのままコピー＆ペーストしてみましょう。

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>はじめてのサイト</title>
</head>
<body>
  <h1>こんにちは！</h1>
  <p>はじめてのホームページです。</p>
</body>
</html>
```

![HTMLを書いている画面](/images/github-pages/html.png)

#### ファイルを保存する

右上の **[Commit changes...]** ボタンを押して変更を保存します。ダイアログが表示されるので、`Commit message` の欄にメッセージ（プロジェクト名などで構いません）を入力し、**[Commit changes]** を押します。

![コミット画面](/images/github-pages/commit.png)

保存されるとファイル一覧画面に戻ります。

:::message
「コミット（Commit）」とは、ファイルの変更を**確定・記録する**操作です。Wordの「上書き保存」に近いイメージで、変更のたびに「いつ・何を変えたか」が記録されます。間違えて変更しても以前の状態に戻すことができます。
:::

### STEP 3: 公開設定を確認する

デフォルトで公開スイッチは入っているはずですが、念のため確認しましょう。

1. リポジトリページ上部の **[Settings]** をクリック
2. 左メニューの **[Pages]** を選択

![Pages設定画面](/images/github-pages/settings01.png)

`Your site is live at...` というメッセージが表示されていればOKです。表示されない場合は2〜3分待ってから再読み込みしてください。

#### GitHub Pagesサイトの公開をON/OFFするには

ここで切り替えるのは「**GitHub Pagesのサイトとしての公開状態**」です。リポジトリ自体はPublicのまま変わらないため、ファイルは引き続き誰でも閲覧できる状態です。サイトとして見せるかどうかだけをON/OFFする設定だと理解してください。

- **Branch** の欄を **`main`** に変更 → サイトとして公開される
- **Branch** の欄を **`None`** に変更 → サイトとしては非公開になる

変更後は **[Save]** をクリックしてください。

![Pages設定画面（非公開時）](/images/github-pages/settings02.png)

### 公開完了！URLを確認しよう

以下のURLにアクセスすると、先ほど入力した `index.html` の内容がWebページとして表示されます。

```
https://(ユーザー名).github.io/(フォルダ名)/
```

スマートフォンからでも見ることができます。表示されない場合は、ユーザー名やフォルダ名を間違えていないか確認してみてください。

## サイトを更新するには？

サイトの内容を変えたいときは、ファイルを編集するだけです。

1. リポジトリで編集したいファイルを選択
2. 右上の **鉛筆アイコン** をクリック
3. 内容を書き換える
4. **[Commit changes]** を押して保存

![ファイル編集画面](/images/github-pages/edit.png)

2〜3分ほどで変更がWebサイトに反映されます。

---

## プロジェクトを追加するには？

プロジェクトを追加するときは、**新しいリポジトリを作る必要はありません**。`(ユーザー名).github.io` リポジトリの中に、プロジェクトごとのフォルダを作るだけです。フォルダ名がそのままURLになります。

| フォルダ名 | 公開URL |
|---|---|
| `quiz/` | `https://(ユーザー名).github.io/quiz/` |
| `animation/` | `https://(ユーザー名).github.io/animation/` |
| `game/` | `https://(ユーザー名).github.io/game/` |

### ファイル構成の例

```
kwaka1208.github.io/
├── index.html         →  /               （トップページ）
├── quiz/
│   └── index.html    →  /quiz/          （クイズプロジェクト）
├── animation/
│   └── index.html    →  /animation/     （アニメプロジェクト）
└── game/
    └── index.html    →  /game/          （ゲームプロジェクト）
```

フォルダを追加するだけでプロジェクトが増やせます。

## まとめ

**GitHub Pages でできること**

- 無料でWebサイトを公開できる
- 自分だけのURL（`https://(ユーザー名).github.io/`）が手に入る

**公開の3ステップ**

1. `(ユーザー名).github.io` という名前のリポジトリを作る
2. フォルダを作成して `index.html` を追加・保存する
3. `https://(ユーザー名).github.io/(フォルダ名)/` にアクセスして確認する

**プロジェクトを追加するには**

同じリポジトリにフォルダを作って、中に `index.html` を入れるだけです。

さっそく、自分だけのサイトを作ってみましょう！
