そもそも、なぜMarkdown Editorを作るというような車輪の再発明的なことをやったのか？世の中にはたくさんのテキストエディタがあって、それらを使えばMarkdownなんて書けるしわざわざ作らなくても？って話だと思うのですが、この記事では私が「Markdown Editor」をあえて作った経緯やnoteには書いていなかった意図などについてまとめています。

## Mardkdownって何？
そもそもMarkdownってなんぞや？って話から始めると、Markdownは見出し、本文、箇条書きなど文章の構造を記号で書き表すルールをまとめたものです。

そして、世の中にはそのルールに従って書くと整形されて出力されるシステムがあります。今みなさんが読んでいるzennもその一つで、zennの記事もMarkdownで書いて公開すると今みなさんが読んでいる記事のようにここが見出しです、ここが本文です、箇条書きには冒頭にマークがついて読みやすくなるようになっています。

今、Markdownで書いた文書の出力に対応したシステムはたくさんあります。Googleドキュメントもその一つで、Markdownで書いた文章をコピーして「編集」メニューの「マークダウンから貼り付け」で貼り付けると見出しなどの書式が反映されるようになっています。その逆も可能で、Googleドキュメントで書いた文章を選んで「マークダウンとしてコピー」を選ぶとMarkdown形式でコピーできるようになっています。

その他にもGitやBacklogなどソフトウェア開発で使われていて、ソフトウェアエンジニアには必須のスキル（これはスキルなんだろうか？）のひとつと言っても過言ではないと思っています。

個人的にはソフトウェア開発以外でも、文章はMarkdownで書くようになっていて、そういうえばiPhoneやiPad、Mac標準のメモアプリやWindows標準のメモ帳も最近Markdownに対応しています。

## 当初は学生向けを意識していた
当初の目的として学生に使ってもらうというのがありました。私はとある専門学校のIT系学科の非常勤講師をしています。IT系学科といえども、そこには今までプログラミングはおろかPCもほとんど使ったことのない人が多いです。そりゃそうです、だから学校に学びにくるのですから、そこは問題ではありません。

そういったこともあり講義では「テキストエディタってのがあってね」というところから始まって、エディタのインストールやGit環境の構築なども最初のうちにやります。最初のうちはフォルダの概念すら知らず自分が保存したデータがどこにあるのかわからなくなってしまう。次の講義の時間では、前の講義で使ったデータがどこにあるのかわからない、そんなことは日常茶飯事です。

こういった学生向けに、アプリのインストールとかアカウントを作るとか面倒なこと抜きに使えるプレビュー機能付きのエディタがあったら便利だなと思ったのが開発のきっかけです。

開発といっても実際に作ったのは生成AIなんですが笑、最初にプレビューを確認しながら編集できるwebベース（それもHTMLファイルひとつで実現できるもの）を作ってと生成AIにお願いしてみました。

プレビュー機能はすぐにできたのですが編集機能がイマイチ。テキストエディタを使ってMarkdownを書いたことがある人ならわかりますが、箇条書きを書いている時に改行すると、次の行に新しい箇条書き項目が出てくるとか、TAB、SHIFT+TABで箇条書きの階層を切り替えられる、といった便利な編集機能はなかったのです。

さすがにそれがないと「Markdownエディタ」とは言えないでしょう。ということで、そこから編集機能の開発をAIと一緒に進めました。開発プロセスの詳細は割愛しますが、細かい動作も含めるとなかなか一筋縄ではいかなかったです。

編集機能のところが一番時間がかったのですが、それでもひとりで開発するよりは圧倒的に速い（というか、ひとりでは開発できなかったと思う）。一日もかからずに、以下の基本的な機能は完成しました。

- 編集画面（ペイン）とプレビュー画面を備える
- 編集操作は一般的なテキストエディタと同じぐらいのアシスト機能を備えている（キー操作も一般的なテキストエディタと同じ操作方法に準拠）
- 作った文書をコピーもしくはダウンロードできる。形式はMarkdown形式と、それを整形したHTML形式

ここまでで、所期の目的は達成していました。

## 複数ファイルを扱えるようにしたい
しかし、この状態ではファイル管理はダウンロードしたものを管理するしかありませんし、うっかりブラウザのウインドウを閉じてしまうと書いたものが消えてしまいます。特に後者は致命的なことになりかねないので、改善しなければなりません。

また、前述の通り学生は自分が書いた文書をどこに置いたか見失うことがあり、それが講義の進行の妨げになることもあるので、できればブラウザの画面の中で作った文書の管理もしたい。

ということで文書の管理機能を追加することにしました。しかしアカウントを作って云々とかいうことはしたくない。全然別の話ですが、アカウントを作るってなった瞬間に年齢制限がかかってしまう（年齢に関係なく誰でも使えるものにしたかったのです）。

そこでアカウントを作ることなく、ブラウザ内でデータ管理できて、うっかりウインドウを閉じてしまってもデータが消えることがない方法ということで、ブラウザのローカルストレージに保存することにしました。このやり方だと異なるPCやブラウザではデータを呼び出せないというデメリットはありますが、アカウントを作らなくていいというメリットを優先しました。

:::message
ちなみに、アカウント作成が必須になると年齢制限がかかる理由については、以下の記事をご覧ください。
https://zenn.dev/kwaka1208/articles/the-laws-and-regulations-on-data-protection

## Markdownは生成AIのプロンプト書きにも使える？
ここまでで、かなり満足できるツールにはなりました。手軽に使えるMarkdown編集ツールとしては十分ではないでしょうか。自分としてもブラウザで動作して過去に作成した文書を呼び出せるツールは欲しかったのでかなり満足していました。

そこで思い出したのは「Markdownって生成AIのプロンプトを書く時にも有効だって話だよね」ということです。その真偽については定かではない（少なくとも私が確認したわけではないという意味で）のですが、「生成AI Markdown」で検索するとそういった情報が出てきますし、生成AIの出力にもMarkdownが使われているので有効なのだろうと思います。

https://www.google.com/search?q=%E7%94%9F%E6%88%90ai+markdown

私自身はプロンプトを常にMarkdownで書いているわけではありませんが、ある程度まとまったタスクを指示する場合はMarkdownで書いています。それはMarkdownで書いた方が生成AIに伝わりやすいという理由ではなく、書いている人間の思考が整理されやすいというのが理由です。

生成AIにタスクを指示した時、一発でうまくいかないことがあります。期待した結果にならなかった時の対処方法としては、

1. 続けて修正の指示を出す
2. 最初の指示内容プロンプトの内容を修正して一からやり直す

の2つがあります。チャット型インターフェースの生成AIのメリットは1のように対話形式で修正を指示できるところなのですが、なぜかどうしても修正指示が通らないことがあります。とっても頑固な人、話のわからない人になったみたいになることがあります（生成AIを使っている人なら経験ありますよね）。こういう時は、修正指示を諦めて一からやり直した方がよかったりします。

この「最初からやり直す」時に指示内容を整理して書いておくと、どこをどのように修正したら良いかが見つけやすく、考えやすいので構造的に書いておくことが有効なんです。

例がないと分かりにくいので、Markdownでプロンプトを書いた例を示します（これもMarkdown Editorに備わっているプロンプトのテンプレートのひとつです）。

```markdown
# 命令
以下の文章を校正・推敲してください。

# 確認事項
- 誤字・脱字がないか
- 日本語として不自然な表現や、回りくどい言い回しがないか
- 文末表現（です・ます調 / だ・である調）が統一されているか
- 【オプション】より魅力的な文章にするための改善案があれば提示すること

# 対象の文章
[ここに文章を入力]
```

このように整理して書いておくと、確認事項にこの項目が不足していたな、とかこの項目は曖昧だったなということを見つけて修正しやすくなるのです。これがMarkdownでプロンプトを書くメリットであると思います。

## プロンプト作成支援機能
長くなりましたが、こういったこともあってぜひ生成AIのプロンプト作成にも使ってもらえればと考えてプロンプトを書くのに便利な機能をつけたい、と考えました。

生成AIの時代ですから、プロンプト作成支援機能といえばそこはやっぱり「AIが手伝ってくれるよね？」という期待が生まれるかもしれません。しかし、このMarkdown Editorは、

- アカウント不要
- インストール不要
- 無料で使える

を目指しているのでAIによる支援機能を追加するのは厳しい（特に無料の部分）。各自でAPIKEY入れてねみたいなやり方もありますが、一般の人にはハードルが高い。初学者にもサクッと使えるものを目指してきたはずなのに、いきなり「APIKEYが〜」と結構な跳躍を求められるのも違う。

などなど逡巡した結果、汎用的に使えるプロンプトのテンプレートを挿入する機能に止めました。しかし、それでは広がらないのでプロンプトの作成をAIに手伝ってもらうためのプロンプト「メタプロンプト」のテンプレート挿入機能も追加しました。

こういった機能を通じて「生成AIへの指示ってこんな風に考えればいいんだ」ということの理解が進めばよいなと考えています。

## 最後に
こんな経緯を経て作ったものを自分たちだけで使うのはもったいない、きっとこういうツールを必要とする人がいるはずだ！（いて欲しい）と考えて公開することにしました。いわゆるAgentic Codingで作ったツールですが、動作確認とコードの確認はしています。

この記事を書いている時点で認識している既知の不具合はiPadで画面の上部が見切れてしまうことです。ブラウザ内で管理している文書の一括ダウンロードや外部のMarkdownファイルの一括取り込みも追加したいなと考えています。

ぜひ、お使いいただければ嬉しいです。

こんな感じで自分に必要なものを自由に作れる時代になりました。生成AIが作ってくれるので、人間は何を作りたいか、どんな機能が欲しいかを自由に考えることができ新たなステージが来た、そしてこれからどうなっていくのだろう？というのが楽しみです。
